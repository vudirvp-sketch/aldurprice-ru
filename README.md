# AldurPrice

> Оверлей проверки цен для Path of Exile 2 (лига «Руны Альдура») с **полной русской локализацией** — clean-room реализация с нуля, оптимизированная под русский клиент игры.

## Что это

**AldurPrice** — десктопное приложение для Windows, которое в реальном времени читает OCR-текстом названия предметов из панели «рунных комбинаций» (runeshape) в Path of Exile 2, ищет актуальные рыночные цены на community-сервисах [poe2scout.com](https://poe2scout.com) и [poe.ninja](https://poe.ninja), и показывает цветные ценники поверх игрового окна рядом с каждой строкой предмета.

Проект создаётся как **clean-room reverse engineering** — реализация с нуля на основе наблюдаемого поведения подобных PoE2-инструментов и анализа публичных данных. Никакой код существующих решений не используется. Архитектура, алгоритмы и UI спроектированы заново с упором на три приоритета: **корректная работа с русским клиентом игры**, **минимальное потребление ресурсов**, **современный UI на WPF .NET 9**.

Приложение ориентировано на русскоязычных игроков PoE2, которые используют официальный русский клиент игры и хотят видеть цены в экономике лиги без переключения языка клиента на английский. Дашборд настроек, оверлей, сообщения об ошибках и чейнджлог — всё на русском языке. Распознавание русских названий рунных комбинаций («Руна огня», «Сплав Рунного отца», «Древняя руна ведьмовства» и т.д.) — первый класс гражданства, не послеthought.

## Ключевые особенности

| Особенность | Реализация |
|---|---|
| **Язык клиента** | RU по умолчанию, ENG как fallback, автоопределение из `game.ini` PoE2 |
| **UI фреймворк** | WPF .NET 9 с MVVM (CommunityToolkit.Mvvm), PerMonitorV2 DPI |
| **Локализация UI** | Полная через `.resx` (русский + английский), переключение без перезапуска |
| **OCR-движок** | Windows Media OCR (primary) + Tesseract 5 (fallback), поддержка обоих с auto-failover |
| **Перевод предметов** | 3 уровня: poe2db.tw (руны) → Exiled Exchange 2 (4319 базовых) → fuzzy matching |
| **Кэш цен** | SQLite persistent (WAL), мгновенный старт, stale fallback при недоступности API |
| **Производительность** | Frame-diffing (пропуск unchanged frames), адаптивный polling 50–200 мс, pause-when-unfocused |
| **Размер сборки** | ~120 МБ (trim + compression, только eng+rus traineddata bundled) |
| **Тесты** | Unit (Core 90%+) + Integration (HTTP mock + real screenshots) + UI (FlaUI) |
| **Лицензия** | MIT, open source |

## Быстрый старт (для пользователей)

Когда проект будет готов, скачать можно будет тремя способами:

1. **Portable ZIP** — распаковать в любую папку, запустить `.exe`. Не требует установки, подходит для флешки.

3. **Из исходников** — `dotnet publish -c Release`, требует .NET 9 SDK.

### Системные требования

- **ОС:** Windows 10 build 1809+ (для Windows OCR), Windows 7+ для Tesseract fallback
- **.NET:** 9.0 Runtime (включён в self-contained сборку)
- **Игра:** Path of Exile 2 с русским языковым пакетом, режим **«Окно без рамки»** или **«Окно»** (эксклюзивный fullscreen не поддерживается)
- **Настройка графики:** «Яркость интерфейса» выше `-0.8` (рекомендуется `0.0` или выше), иначе OCR не распознает текст
- **Дополнительно:** интернет для запросов цен (обновление каждые 15 минут, кэш локальный)

## Как это работает

Приложение работает в три параллельных слоя, которые общаются через потокобезопасный in-memory кэш поверх SQLite:

1. **Слой захвата (Capture)** — каждые 50–200 мс (адаптивно) захватывает регион экрана с панелью рунешейпов через `PrintWindow` или `Windows.Graphics.Capture`. Детектор панели анализирует яркость верхней строки, чтобы понять, открыта ли панель, и не тратить CPU, когда панель закрыта. Frame-differ на основе perceptual hash пропускает обработку идентичных кадров.

2. **Слой распознавания (OCR)** — захваченный битмап проходит через препроцессор (бинаризация + цветовая фильтрация под цвет текста игры), затем через Windows Media OCR (быстро, нативно) или Tesseract 5 (fallback). Распознанный текст нормализуется, из него извлекаются названия предметов и количества (с поддержкой русского префикса «шт» и trailing-числа «Предмет 1»).

3. **Слой цен (Pricing)** — каждые 15 минут фоновый воркер обновляет кэш цен с выбранного источника (poe2scout или poe.ninja). Названия предметов переводятся на английский через `ItemNameTranslator` (используя `rus.ndjson` из Exiled Exchange 2 и кастомный маппинг рунных комбинаций из poe2db.tw), затем ищется точная цена, при неудаче — fallback по категориям уников, tier-fallback для GREATER/PERFECT орбов, и fuzzy-matching против известных ключей цен.

Результат рендерится в WPF-оверлее с цветовой кодировкой: красный (дёшево), оранжевый (средне), зелёный (дорого). Пороги настраиваются или вычисляются автоматически по самой дорогой позиции в скане.

## Атрибуция и лицензия

Проект распространяется под лицензией **MIT** (см. [LICENSE](LICENSE)).

В проекте используются данные и библиотеки сторонних разработчиков:

- **[Exiled Exchange 2](https://github.com/Kvan7/Exiled-Exchange-2)** — данные перевода названий предметов (`items.ndjson`, сгенерированные из `BaseItemTypes.json` PoE2). Используются под их лицензией — см. `ocr/translations/LICENSE`.
- **[poe2db.tw](https://poe2db.tw/ru/Runeshape_Combinations)** — источник русских названий рунных комбинаций и ремнантов лиги «Runes of Aldur». Fan-wiki, данные публично доступны.
- **[Tesseract OCR](https://github.com/tesseract-ocr/tesseract)** — OCR-движок (fallback), Apache 2.0.
- **[poe2scout.com](https://poe2scout.com)** — источник цен с 24-часовым усреднением.
- **[poe.ninja](https://poe.ninja)** — альтернативный источник цен.

Все торговые марки принадлежат их владельцам. AldurPrice не аффилирован с Grinding Gear Games, poe2scout, poe.ninja или poe2db.tw. Инструмент создан сообществом для сообщества.
